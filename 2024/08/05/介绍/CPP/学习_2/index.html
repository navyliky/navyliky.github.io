友元

在类的定义中，通过对类的成员添加访问限定来控制在类的外面访问类的成员；通过友元的指定，可以在外部特定的函数、外部类的某些成员函数或者外部的类中访问收限制的成员
友元是一种在类的外部访问类中受限的成员的一种方法，在C++中友元的定义使用关键字friend

a. 类外的普通函数成为类的友元函数 – 普通函数中可以直接访问(通过对象)对象的所有成员

b. 类外的其他类中的某些特定的成员函数成为类的友元函数 – 这些说明为友元的成员函数中可以直接访问对象的所有的成员

c. 其他类成为当前类的友元类 – 在类的所有的成员函数中都可以访问友元类中的成员

** 注意: 友元关系的指定是单向的；Student类中说明Teacher类是友元类仅表示Teacher类是Student类的友元，在Teacher类的所有成员函数中都可以访问Student类的成员；但Teacher类没有指定Student类为友元，所以在Student类的成员函数中不能访问Teacher类的私有成员。 **

继承

从已有的类实现新类的方式，通过继承，新类具有原来已有的类的所有属性和行为
继承性是面向对象程序设计支持代码重用的重要机制。面向对象程序设计的继承机制提供了无限重复利用程序资源的一种途径。通过C++语言中的继承机制，一个新类既可以共享另一个类的操作和数据，也可以在新类中定义已有类中没有的成员
在继承关系中，
原来的类 新的类
基类 派生类
父类 子类

C++中的继承方式: 单继承，多继承
单继承: 由一个唯一的基类派生出一个新的类的继承方式
多继承: 由多个基类共同派生出一个新的类的继承方式

单继承
class 类名 : 派生/继承方式 基类名
{
//类的实现: 派生类自己独有的属性和成员函数的实现
};
派生/继承方式: private, protected, public

通过继承，新的类拥有基类的一切:

不同继承方式下成员访问权限的变化:
private：私有的，只能在类的成员函数中访问
protected: 保护的，可以在类的成员函数和派生类的成员函数中访问
public: 公有的，可以在类的成员函数、派生类的成员函数、类外的函数中都可以使用

基类访问方式 派生方式
public protected private
private 不能访问 不能访问 不能访问
protected protected protected private
public public protected private
派生类对象可以看作是包含了一个基类对象，在基类对象的基础上加上派生类对象自己独有的部分构成的

重写:

在派生类中对于基类的成员进行重写实现称为重写


重写(overwrite)：在派生类中重写基类的成员以后，通过基类对象访问的是基类中实现的版本，通过派生类对象访问的是派生类中重写以后得版本；派生类重写以后的实现覆盖了从基类继承的版本；通过派生类对象不能直接访问被覆盖的从基类继承的版本，可以通过明确指定的方式来访问: 派生类对象.基类名::覆盖的函数调用

运行结果:
派生类对象调用覆盖的基类继承下来的版本:
Base::
p_num = 0
t_num = 0
r_num = 0

重载(overload)

在派生类中对基类的成员进行重载(函数原型不同)，这时派生类中重载的版本会隐藏从基类继承的版本，通过派生类对象只能够访问到派生类中重载以后得版本，被隐藏的从基类继承的不能直接访问，可以通过明确指定的方式来访问: 派生类对象.基类名::隐藏的函数调用

基类和派生类的关系:

  1. 派生类是基类的具体化。
  2. 派生类是基类定义的延续。
  3. 派生类是基类的组合。 派生类对象在实例化的时候会先调用基类的构造方法(默认调用的是基类的无参构造)构造派生类中从基类继承的部分(派生类对象构造的时候先调用基类构造方法构造一个基类对象)， 然后再调用派生类构造方法构造自己独有的部分
    派生类对象在析构的时候，先调用自己的析构方法释放自己独有的部分，再调用基类的析构方法释放基类对象

在派生类的构造函数的初始值表中可以显式指定要调用的基类构造函数的版本

注意: 在派生关系中，基类的构造方法和析构方法不能被派生类继承，而是C++使用一种机制来使得派生类对象的构造过程中调用基类的构造方法初始化派生类中的基类对象，在析构的时候调用基类的析构方法来释放基类对象。
